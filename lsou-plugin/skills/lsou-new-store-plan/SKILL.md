# 来搜新店运营规划

## 一句话触发

用户说“生成新店运营规划”“看新店 SOP”“新店怎么启动”“导出执行报告”时使用。

## 页面与工具

- URL: `https://test.l-sou.cn/operationsCOTNT/new-store-plan.html`
- 推荐子代理: `browser`
- 浏览器工具族: `browser_*`
- 主动作按钮: 优先读取表格；用户要求导出时点击 `导出执行报告`

## 教程图

- 回复中默认展示：`![来搜新店运营规划教程图](resources/tutorial-images/08-lsou-new-store-plan.png)`。
- 若用户明确只要数据、快速结果或不要图片，可以省略教程图。

## 前置检查与异常处理

- 使用前提醒用户：需要已安装并启用来搜Plugin、已登录来搜账号、并已绑定要规划或分析的店铺。
- 打开页面后先检查登录、授权、店铺绑定和权限状态；异常时停止并请用户处理。
- 如果用户没有说明要“查看规划”还是“导出报告”，优先用 `Ask User` 询问；默认推荐“查看规划并总结”。
- 如果 SOP 表为空、负责人/日期控件无内容或页面白屏，刷新或重试一次；仍无数据时报告页面状态。
- 导出报告属于生成文件动作，只有用户明确要求导出时才点击。

## 实测边界

- 实测为静态新店 SOP 模板，未观察到真实保存、提交或后台请求。
- 表格含负责人输入、截止日期和状态下拉，但没有稳定保存动作；不要声称已保存任务。
- `导出执行报告` 实测更像前端提示，不要说已生成真实 PDF 文件。
- `分享进度到微信` 为模拟弹窗/二维码；没有用户明确要求不要点击。
- 页面进度和表格状态可能不一致；输出时标注为页面当前显示。
- 部分阶段链接可能 404 或占位，不作为稳定流程。

## 执行步骤

1. 打开 URL。
2. 读取新店运营阶段表。
3. 重点抓取序号、运营阶段、对应 SOP、落地执行项目、建议天数、负责人、截止日期、当前状态、自动化动作。
4. 如果用户要求报告或文件，再点击 `导出执行报告`。

## 输出

输出新店启动路线图模板，按阶段列当前进度、卡点、责任人和下一步。明确这是页面 SOP 模板，不是后台任务系统。
