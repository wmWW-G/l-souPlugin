# 来搜店铺深度诊断

## 一句话触发

用户说“做店铺诊断”“做店铺深度诊断”“看全店问题”“出一份店铺经营诊断”时使用。

## 页面与工具

- URL: `https://test.l-sou.cn/diagnosisSKILL/store-diagnosis.html`
- 推荐子代理: `browser`
- 浏览器工具族: `browser_*`
- 主动作按钮: `同步数据`

## 教程图

- 回复中默认展示：`![来搜店铺深度诊断教程图](resources/tutorial-images/02-lsou-store-diagnosis.png)`。
- 若用户明确只要数据、快速结果或不要图片，可以省略教程图。

## 前置检查与异常处理

- 使用前提醒用户：需要已安装并启用来搜Plugin、已登录来搜账号、并已绑定要分析的店铺。
- 打开页面后先判断是否出现登录、授权、验证码、未绑定店铺、无权限或请选择店铺；出现这些状态时停止读取，让用户先处理。
- 若用户没有说明按月、按周还是按日，优先用 `Ask User` 弹出选择；默认推荐“按月”，适合店铺月度诊断。
- 点击同步后如果诊断模块仍为空、显示 `-`、暂无数据或只有表头，等待后重试一次；仍无数据时报告页面状态。
- 如果某些模块有数据、某些模块无数据，只输出已看到的数据，并标注缺失模块。

## 实测边界

- 实测核心五维指标、子类目、国家市场和执行计划可见；产品、营销或品广部分表格可能为空或只有表头。
- 页面标题里的月份可能与时间选择器不一致；必须以用户选择和页面当前选择器为准，并提示这个不一致。
- 切换月/周/日后，如果没有重新同步或内容未变化，不要声称已得到对应周期的新诊断。
- 部分表为空不是整页失败；逐模块说明“有数据/无数据/未展示”。

## 执行步骤

1. 打开 URL。
2. 按用户要求设置月份、周或日期；无要求则使用页面默认时间。
3. 点击 `同步数据`，等待诊断模块返回。
4. 读取核心五维运营指标、子类目穿透诊断、产品参谋诊断、营销中心诊断、品广中心诊断、国家市场诊断。
5. 读取页面底部落地执行计划，保留责任人、时间和检查人字段。

## 输出

输出“最严重的问题、原因、证据、整改动作”。表格很长时只保留 TOP 项、异常项和执行计划，并标注空表模块。
